# Second Brain OS — Canon Declaration

> Per SIP Layer 1. This file declares SBO's canon posture.

## Decision

**SBO declines canon at the substrate layer.**

SBO is a vertical that composes SIP. It defines a personal-knowledge architecture (two vaults, dual-write ingestion, agent contracts) — it does not define archetypes, world rules, or domain constants.

## Why

Per SIP § Sovereignty clause: substrate declines vertical canon. SBO is itself a vertical from SIP's perspective, but it serves as a substrate for downstream installations. Per the same recursive rule, SBO declines to impose canon on its adopters.

## What this means for adopters

You may bring your own canon. Common patterns:
- A personal "operating manual" file at `brain/_meta/operating-manual.md`.
- A values doc at `brain/_meta/values.md` (the paid `ikigai-extractor` agent writes here).
- A relationships taxonomy at `brain/_meta/relationships-taxonomy.md`.

None of these are required for SBO compliance. Vault structure (folders + write zones) is required; their contents are sovereign.

## Composition rules

If you adopt Arcanea canon (Guardians, Vel'Tara, Hz grounding), do so via `/luminor-board` instead of `/starlight-board` and accept the CC-BY-NC attribution that comes with Arcanea canon.

This declaration is the SBO-vertical canon posture. Per SIP § Sovereignty clause.
